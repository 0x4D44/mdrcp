# 2026.06.19 - JRN - autobin-aware bin discovery (mdrcp)

## Context
mdrcp deploys only the package-named binary because `manifest_bin_names`
hand-parses `[[bin]]` from Cargo.toml and falls back to the package name,
ignoring Cargo autobins. deltic (no `[[bin]]`, four autobins) → only `deltic.exe`
reaches `C:\apps`; `delticw.exe` is dropped. Root cause of the single-binary
console-window bug investigated in the deltic repo.

User picked "fix mdrcp itself" (cargo metadata discovery) over a deltic-side
`[[bin]]` workaround.
